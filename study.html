科目一：人工智慧基礎理論

1-1 人工智慧概念

  • 負責任的 AI： 開發者對系統行為負責。
  • 監督機制 (Supervision)：
    • 人在迴圈內 (HITL)： 每步皆須人類批准（高風險、高準確要求）。
    • 人在迴圈上 (HOTL)： 人類日常監督，必要時干預。
    • 人在迴圈外 (HOOTL)： 系統完全自主決策，人類不即時干預。
  • 法規： 2025年《人工智慧基本法》參考歐盟 監理沙盒 (Regulatory Sandbox)。
  • 能力與電腦視覺 (CV)：
    • 弱 AI (Narrow AI)： 專精單一任務 (如語音辨識、瑕疵檢測)，目前主流。其模型設計侷限於特定領域，無法直接遷移。
    • 強 AI (General AI / AGI)： 具備人類水準理解力、跨領域學習與推理能力 (目前尚在理論階段)。
    • 超級 AI (ASI)： 智力在各方面（含科學創新、通識與社交能力）都大幅超越最聰明人類大腦。
    • 影像分類： 判斷影像類別（如：是否有煙霧）。
    • 物件偵測： 找出位置並用方框 (Bounding Box) 標示（如：標出煙霧位置）。
    • 影像分割： 精確標示像素級邊緣（如：精確標示煙霧的輪廓）。
  • 可解釋 AI (XAI)：
    • 後處理 (Post-hoc)： 如 LIME，專門解釋黑盒模型中「單一局部預測」的決策過程。
    • SHAP (沙普利加性解釋)： 基於賽局理論，能精確提供「在單一預測結果中，各輸入特徵對最終輸出所產生的貢獻分配」。
    • 顯著性圖 (Saliency Map)： 常用於影像模型，標示出對預測結果影響最大的像素區域。
    • 反事實解釋： 分析「若改變某特徵，結果會如何變化」。
  • 模型評估與維運：
    • 指標選擇： 目標為「不能漏掉異常」（如疾病篩檢、停機偵測）時，應看重 召回率 (Recall)。
    • 資料偏移 (Data Drift)： 上線後準確率下降通常是因為使用者行為或統計特徵隨時間改變，而非模型架構損壞。
  • 金融 AI 指引： 若模型偏誤，啟動 人工覆核。揭露條款通常排除「模型原始碼」。
  • 維運實務挑戰：
    • RAG 系統： 應採用增量更新 (Incremental) 避免停機。
    • 批次推論 (Batch Inference)： 挑戰在於 GPU 資源、吞吐量與延遲管理，而非訓練資料標註品質。
    • GPU 利用率瓶頸： 若 GPU 運算慢但負載低，通常是 I/O 瓶頸（硬碟搬移至 GPU 太慢）。
    • 物聯網應用： 智慧城市監測（如 PM2.5、設備狀態）主要依賴 感知器網路 (Sensor Network)。

1-2 資料處理與分析概念

  • 大數據特性 (5V)： Volume (量)、Velocity (速)、Variety (多樣性)、Veracity (真實性)、Value (價值：最終目的)。
  • 資料型態：
    • 結構化： 資料庫紀錄，易於計算。
    • 半結構化： JSON、XML 標註資料。
    • 非結構化： 錄音、影像、文字對話，占比最高且處理難度最大。
  • 資料工程：
    • ETL 流程： E (擷取) -> T (轉換：清洗、格式統一、處理冗餘) -> L (載入)。
    • ELT： 先載入雲端數據湖再轉換，適合大數據處理。
  • 分析類型：
    • 探索性分析 (EDA)： 繪製圖表尋找潛在關係。
    • 預測性分析 (Predictive)： 推算未來趨勢。
  • 資料品質： 冗餘 (Redundant) 指語意重複；重複 (Duplicate) 指紀錄完全一致。缺失值可刪除或填補平均值。
  • 敘述性統計指標：
    • 平均數： 易受極端值（離群值）影響。
    • 中位數： 對離群值具穩健性。
    • 標準差/變異數： 越大代表波動越大、品質越不穩定。
  • 視覺化與轉換：
    • 直方圖 (Histogram)： 觀察連續資料的分佈、偏態與多峰性。
    • 散佈圖 (Scatter Plot)： 觀察兩個數值變數間的相關性（如身高與體重）。
    • 對數轉換 (Log Transform)： 處理高度偏態資料，使其接近常態分佈。

1-3 機器學習理論

  • 學習模式：
    • 監督式： 垃圾郵件過濾、房價預測 (需標籤)。常用演算法如：決策樹、隨機森林、貝氏分類器 (Naive Bayes) (極適合文字分類與郵件過濾)。
    • 非監督式： 在無標籤下探索隱含模式，如客戶分群 (Cluster)、降維 (PCA)。適合用於「缺乏完整異常標註」的設備狀態檢測。
    • 半監督式： 結合少量標註與大量未標註資料。
    • 自監督式： 利用資料本身的結構產生目標（如 Masked LM），大幅降低對人工標註的依賴。
    • 強化學習 (RL)： 透過環境獎勵/懲罰優化策略。適用於圍棋、自駕車、自動交易系統。
    • RLHF (人類反饋強化學習)： 當大型語言模型 (LLM) 需優化回覆偏好並遵循規範時，導入人類回饋以確保行為符合期望。
    • 遷移學習 (Transfer Learning)： 利用既有預訓練模型微調，適用於標註樣本稀少且成本高昂的任務。
  • 重要概念：
    • 偏差與變異權衡 (Bias-Variance Tradeoff)： 在模型欠擬合 (Bias) 與過擬合 (Variance) 之間取得平衡。
    • 資料去偏 (Data Debiasing)： 透過調整或擴充訓練語料，平衡不同群體比例，減少模型偏見。
  • 過擬合 (Overfitting) 解決方案：
    • 增加數據量、使用 交叉驗證。
    • 正則化 (Regularization)：
      • L1 (Lasso)： 產生稀疏解 (強迫不重要特徵權重歸零)，具特徵選擇功能。
      • L2 (Ridge)： 縮小權重但不歸零，使模型更平滑。
    • Dropout (隨機失活節點)、資料增強 (旋轉翻轉)。
  • 核心演算法：
    • K-means (分群)： 依賴距離計算，易受雜訊與離群值 (Outlier) 影響，且無法直接處理類別型資料。
    • KNN (分類)： 基於鄰近樣本的多數決 (Majority Vote)。
    • 邏輯迴歸 (Logistic)： 預測機率 (0~1)，適用於二分類 (如罹病風險)。
    • 隨機森林 (Random Forest)： 處理多特徵、提升穩健性與泛化能力。
    • PCA (主成分分析)： 降維技術。將特徵轉換為新變數，主要目的是「減少輸入特徵維度以提升運算效率」，同時保留最大的資料變異資訊量。
  • 神經網路架構：
    • CNN： 擅長處理具「網格空間結構」資料，捕捉局部特徵 (影像辨識、物件偵測及分割)。
    • RNN/LSTM： 專注於具「時間相依性」的序列資料。RNN 常發生 梯度消失 難以保留早期資訊；LSTM 為解決此問題而生。
    • Transformer： 透過 自注意力機制 (Self-Attention) 平行運算並捕捉全局關聯，解決 RNN 無法處理長文本的痛點，是當今大型語言模型的底層基石。
    • 寬深模型 (Wide & Deep)： 同時記憶特徵組合規律並具備泛化預測能力。
    • 啟動函數：
      • ReLU： 可改善梯度消失問題，提升訓練穩定性。
      • Sigmoid： 用於二分類或門控機制（如 LSTM 的忘記門）。
    • 損失函數： MSE (迴歸/房價預測)；交叉熵 (分類)。
  • 資料處理與特徵工程：
    • One-hot Encoding： 用於無序類別（如地區、交通工具）。
    • Ordinal Encoding： 用於有序類別（如會員等級、滿意度）。
    • 數值標準化/正規化： 梯度下降模型必做，防止權重因數值範圍不同而偏斜。
    • SMOTE： 針對類別極端不平衡（如詐騙偵測），在特徵空間中合成 (Synthetic) 新的少數類樣本。
    • PEFT (如 LoRA)： 僅更新極少量參數進行微調。
  • 搜尋演算法：
    • 二分搜尋 (Binary Search)： 適用於已排序陣列，每次排除一半數據，效率為 $O(\log n)$。

1-4 鑑別式 vs 生成式 AI

  • 核心原理：
    • 鑑別式 (Discriminative)： 學習邊界，專注於分類與預測。
    • 生成式 (Generative)： 學習資料分佈，創造相似的新結果。
  • 生成模型三大支柱與自迴歸：
    • GAN (生成對抗網路)： 透過「生成器」產出假資料，並由「判別器」分辨真偽，兩者相互博弈訓練，適合生成高度逼真的人臉或假影像。
    • VAE (變分自編碼器)： 透過 潛在空間 (Latent Space) 重建資料，用於風格一致的新樣本生成或異常偵測。
    • Diffusion (擴散模型)： 透過逐步在資料中「加入雜訊」再訓練模型「去除雜訊」來還原並生成全新資料，視覺品質最高、細節穩定 (如 Midjourney、Sora)。
    • AR (自迴歸模型)： 根據前一個字預測下一個字，GPT 系列、文本生成基礎。
  • Transformer 與 LLM 效能優化：
    • 計算瓶頸： 自注意力機制需求隨序列長度呈 平方級 ($O(n^2)$) 增長。
    • Flash Attention： 優化 IO 存取，提升速度、資源效率並減少記憶體佔用。
    • 剪枝 (Pruning)： 移除影響較小的權重參數，實現模型輕量化並提升效率。
    • 批次處理 (Batching)： 提升 GPU 使用率 與 吞吐量 (Throughput)，但可能增加單筆請求的 延遲 (Latency)。
    • KV Cache： 用於加速推理，避免重複計算已生成的 token，但會增加記憶體壓力。
  • 輸出層： Softmax 函數 將輸出轉為機率分佈 (總和為 1)，常用於多分類推薦系統。

科目二：人工智慧應用實務

2-1 No Code / Low Code 與 Agentic AI 概念

  • 平台特性： No-Code 適合快速建立應用；Low-Code 具強整合性與自訂性。
  • 維運風險： 影子 IT (Shadow IT) 指未經管理的人員自行建立應用；數據漂移 (Data Drift) 指統計特徵隨時間改變。
  • Agentic AI 核心：
    • 高度自主性： 不僅能推斷答案，還能在 解決方案圖譜 (Solution Graph) 上，自主利用 廣度優先 (BFS)、深度優先 (DFS) 或最佳優先 (Best-first) 等演算法進行探索，尋找最佳解決路徑。
    • MCP 協議： AI Host -> MCP Client -> MCP Server 標準化連結外部工具。
    • A2A： Client Agent 發起，Remote Agent 執行回傳。
    • 組件： FAQ 用 向量檢索；動作執行用 API 調用。

2-2 生成式 AI 應用領域與工具使用

  • 核心概念：
    • 多模態模型 (Multimodal)： 能同時處理與生成 文字、圖片、影片 等多種媒介內容。
  • 提示工程 (Prompt Engineering) 進階策略：
    • Zero-shot / Few-shot： 零樣本或提供範例。風險：範例不足會導致 領域偏移 (Domain Shift)。
    • Chain-of-Thought (CoT)： 要求「逐步說明推理過程」，提升數學與邏輯準確性。
    • Tree of Thoughts (ToT)： 探索多路徑並比較方案，適合複雜決策任務。
    • Graph Prompting： 用於捕捉非線性、網絡化資訊或多條件關聯分析。
  • 檢索增強生成 (RAG) 實務：
    • 核心目的： 檢索外部資料庫以獲取最新資訊，降低 幻覺 (Hallucination)。
    • 文本切分 (Chunking)： 將文件切為小片段以 提高檢索相關性 並減少噪音。
    • RAG vs. Fine-tuning： 微調 用於調整風格、語氣或學習格式；RAG 用於動態更新知識。
  • 參數與評測：
    • 溫度 (Temperature)： 低溫 (0.1-0.3) 輸出穩定、可預測；高溫 (1.0+) 增加創意。
    • 基準測試 (Benchmarks)： TTQA (台灣本土知識)；MMLU (多領域理解)。
  • 安全性與內容溯源：
    • SynthID： Google 開發，在影像每一幀嵌入 「不可見」數位浮水印。
    • C2PA (Content Credentials)： 在元數據中嵌入來源證明，追蹤內容是否由 AI 生成。
    • 安全性： 防範 提示攻擊 (Prompt Injection)。

2-3 生成式 AI 導入與規劃

  • 導入生命週期與架構：
    • 標準流程： 1.需求確認/目標定義 -> 2.PoC 驗證 -> 3.上線部署 (Deployment) -> 4.監控與優化 (Monitoring) -> 5.價值擴散 (Value Scaling)。
    • MVP 與安全性： 快速開發 (如 Vibe Coding) 後，正式上線前必須經過重構與安全測試。
    • 高可用性： 採用 負載平衡 (Load Balancing) 確保架構穩定。
  • 成本與投資報酬 (ROI)：
    • Token 經濟學： 考量 API 費率與輸入/輸出 Token 數。AI API 成本（如 960 元）對比人工（如 30,000 元）具高 ROI。
  • 風險管理與法規：
    • 歐盟 AIA 風險分級：
      • 不可接受風險： 絕對禁止。如 社會信用評分、遠程生物辨識執法。
      • 高風險： 嚴格監管。如 醫學影像診斷、放貸審核。
      • 有限/低風險： 需符合透明性要求（標註為 AI 生成）。
    • 風險應對策略： 外包安全管理屬 風險轉移 (Risk Transfer)。
    • 偏差檢測 (Bias Detection)： 屬於檢測措施；重新加權資料屬於緩解措施。
  • 台灣 AI 評測指標 (數位發展部 adi)：
    • 四大核心： 1.當責性、2.可靠性、3.隱私及資安、4.透明性 (互動性非核心維度)。
  • 資安與隱私技術：
    • Zero Trust (零信任架構)： 確保數據使用安全性。
    • Zero-Retention (零保留)： 避免第三方 API 記憶敏感資訊。
    • 同態加密 (Homomorphic Encryption)： 讓資料在加密狀態下仍能進行模型運算，保障高度隱私。
    • 資料原則： 優先採取 「資料最小化」 與 「去識別化」。
    • 聯邦學習 (Federated Learning)： 在隱私前提下整合跨部門敏感資料進行訓練。
  • 優化技術與挑戰：
    • Reinforcement Fine-tuning (RFT)： 透過 獎勵訊號 調整回應行為偏好。
    • 災難性遺忘 (Catastrophic Forgetting)： 在微調時過度適應新資料，導致模型忘記原本學到的廣泛知識。
  • 基礎設施： 私有化部署符合數據主權；需 GPU 資源與彈性儲存空間。
